1. PROJECT PHILOSOPHY & SCOPE PIVOT
Vandir is a 50,000 x 50,000 block Minecraft world (Y -64 to 448). This is a terrain translation pipeline, not a procedural world generator. Source heightmaps and masks from Gaea are translated 1:1 into Minecraft chunk data.
CRITICAL SCOPE UPDATE: The world is an untouched wilderness. There are NO human structures, NO roads, and NO ruins. All legacy code or logic pertaining to narrative structures is deprecated. The schematic placement system is strictly for geological and ecological features (custom trees, fallen logs, glacial erratics, karst pillars, scree fields).
The standard for quality: The pipeline must think globally and render locally. Every biome, cliff face, and river bend must have the internal logic of geology and climate behind it.
2. CORE CONSTRAINTS & MAGIC NUMBERS
 * World Size: 50,000 x 50,000 blocks (97 x 97 tiles).
 * Tile Size: 512 x 512 blocks.
 * Vertical Range: Y -64 to Y 448 (vandir_height.zip datapack).
 * Sea Level: raw 17050 -> MC Y 63.
 * DataVersion: 4556 (MC 1.21.10).
 * Python Environment: pythoncore-3.14-64\python.exe (rasterio, nbtlib, PyQt6, scipy, Pillow).
 * Height Polarity: HIGH raw 16-bit = HIGH terrain. Normalization: height_norm = raw / 65535 (0.0 = ocean floor, 1.0 = peak). Sea norm is approx 0.260.
 * Override Upscaling: ALWAYS use Image.NEAREST. Bilinear interpolation creates phantom biomes. ALWAYS np.fliplr() override source.
 * Chunk Writing: Biome PalettedContainer min_bits=1 (NOT 4). isLightOn=0. Fluid ticks on top water block only.
3. PIPELINE ARCHITECTURE (THE PRE-BAKE IMPERATIVE)
The pipeline is designed to eliminate tile-boundary seams by pre-calculating all spatial data globally. No spatial math relying on neighboring pixels (like EDT) is allowed during chunk generation.
 * Stage 0 (Geodata Manifest): Single affine transform matrix bridging Gaea and Minecraft coordinates.
 * Stage 1 (Global Pre-Bakes): Derived masks (slope, aspect, shore_distance) are generated globally via scipy/rasterio and cached as TIFs.
 * Stage 2 (Global Biome Assignment): override.tif -> Terrain Class -> Hydrology -> Patch Noise -> Final Biome String.
 * Stage 3 (Flora & Geology Plan): Global Poisson Disk Sampling masked by slope, biome, and Simplex noise. Saves to flora_geology_plan.json.
 * Stage 4 (Chunk Generation): Embarrassingly parallel. Reads pre-baked slices.
 * Stage 5 (Writer): Asynchronous nbtlib MCA writer to prevent disk I/O bottlenecks.

from scipy.ndimage import distance_transform_edt
def generate_global_shore_distance():
with rasterio.open('masks/height.tif') as src:
height_data = src.read(1)

C. Global Flora & Geology Planner
Issue: Random per-tile placement creates clumping and edge-slicing.
Fix: Implement Bridson's algorithm via scipy to pre-bake organic coordinates.
Code:
import numpy as np
import rasterio
from scipy.stats import qmc
from core.noise_fields import generate_simplex_noise
class GlobalFloraPlanner:
def init(self, world_size=50000, chunk_size=5000):
self.world_size = world_size
self.chunk_size = chunk_size
def generate_flora_plan(self, radius=5.0, noise_scale=0.005, clearing_thresh=0.3):
valid_points = []
with rasterio.open('masks/slope.tif') as src_slope:
for x_start in range(0, self.world_size, self.chunk_size):
for z_start in range(0, self.world_size, self.chunk_size):
engine = qmc.PoissonDisk(d=2, radius=radius/self.chunk_size, seed=42+x_start)
pts = engine.fill_space() * self.chunk_size
gx = (pts[:, 0] + x_start).astype(int)
gz = (pts[:, 1] + z_start).astype(int)
mask = (gx < self.world_size) & (gz < self.world_size)
gx, gz = gx[mask], gz[mask]
window = rasterio.windows.Window(x_start, z_start, self.chunk_size, self.chunk_size)
slope_data = src_slope.read(1, window=window)
lxi = np.clip(gx - x_start, 0, self.chunk_size - 1)
lzi = np.clip(gz - z_start, 0, self.chunk_size - 1)
geo_mask = slope_data[lzi, lxi] < 0.60
gx, gz = gx[geo_mask], gz[geo_mask]
noise = generate_simplex_noise(gx, gz, scale=noise_scale, seed=99)
org_mask = noise > clearing_thresh
fx, fz = gx[org_mask], gz[org_mask]
for x, z in zip(fx, fz):
valid_points.append({"x": int(x), "z": int(z), "type": "tree"})
return valid_points
